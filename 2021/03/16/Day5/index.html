Java面试题之Java基础总结（二）

发表于 2021-03-16 更新于 2021-04-25 分类于 面试题

1.Q：Java的反射机制是什么？

Java反射机制是在运行状态下，对于任意一个类，都能知道这个类的所有属性和方法，对于任意一个对象，都能调用其任意方法和属性。简而言之，这种动态获取信息以及动态调用对象方法的功能称为Java语言的反射机制。

运用实例：（1）Spring框架以配置文件的形式配置各类Bean，Spring容器会根据我们的需求动态加载所需Bean。

（2）根据类名创建实例，类名从配置文件读取，不用new，从而达到解耦效果。如需要实例化某个类，使用Class.forName(className).newInstance()。

2.Q：Java的IOC是什么？

IOC（inversion of control）即控制反转，把设计好的类交给系统控制，而非在类的内部控制。

【计组】3.Q：进程间通讯的七种方式？

  • 管道pipe：半双工的通信方式，数据只能单向流动，且只能在具有亲缘关系的进程间使用。
  • 命名管道FIFO：半双工，但允许无亲缘关系进程间的通信。
  • 消息队列MessageQueue：由消息链表存在放内核中，由消息队列标识符标识。克服了信号传递少、管道只能承载无格式字节流及缓冲区大小受限等缺点。
  • 共享存储SharedMemory：映射一段能被其他进程所访问的内存。是最快的IPC方式，通常与信号量配合使用，实现进程间的同步与通信。
  • 信号量Semaphore：信号量是一个计数器，控制多个进程对共享资源的访问。通常是一种锁机制。
  • 套接字Socket：是一种进程间通信机制，可用于不同进程间的进程通信。
  • 信号signal：用于通知接收进程某个事件已经发生。

【OS】4.Q：操作系统的安全状态是什么？

安全状态指的是系统能按照某种顺序来为每个进程分配资源，使每个进程可以顺序完成。该顺序称为安全序列。如果系统不存在这样的安全序列，则称系统处于不安全状态。

运用示例：银行家算法是一种代表性的避免死锁的算法。虽然允许进程动态地申请资源，但系统在进行资源分配之前，应该先计算此次分配资源的安全性，如果不会导致系统进入不安全状态，则分配，否则等待。

5.Q：Spring是什么？

Spring是一个轻量级的开源框架，同时也是一个三层架构，为J2EE应用程序开发提供集成的框架。

  • 三层架构：Web层（Spring MVC）；业务层（Spring IOC）；持久层（Spring JDBC、ORM等）；
  • 核心：控制反转IOC和面向切面编程AOP。
  • 对于Spring中的IOC，Spring中最高级的抽象是BeanFactory，对应工厂模式，分为两种模式——单态和原型。

6.Q：AOP是什么？

AOP（aspect oriented programming）面向切面编程是OOP的补充和完善。OOP虽然允许开发者定义纵向的关系，但并不适合定义横向的关系。如日志功能，日志代码往往横向散布在所有对象层次中，而与它对应的对象的核心功能毫无关系。这些代码被称为横切，导致大量代码的重复。

而AOP使用横切的技术将影响到多个类的公共行为都封装到一个可重用模块，从而减少代码的复用。

7.Q：String str = new String(“abc”)创建了几个对象？

如果常量池中没有”abc”的话，两个，一个是常量池中的”abc”字符串对象，一个是堆中的String对象。

①String str = “abc”

首先在常量池中查找是否存在内容为”abc”的字符串对象。

如果不存在就在常量池中创建”abc”，并让str引用该对象。

如果存在就直接让str引用该对象。

②String str = new String(“abc”)

首先在堆中创建一个指定对象”abc”，并让str引用指向该对象。

在字符串常量池中查看，是否存在内容为”abc”的字符串对象。

如果存在，将new出来的字符串对象与字符串常量池中的对象联系起来。

如果不存在，则在字符串常量池中创建一个内容为”abc”的字符串对象，并将堆中的对象与之联系起来。

注：字符串池在堆中。JVM的堆中存放对象的实例，栈中存放存储对象的引用。

9.Q：静态变量和实例变量的区别

静态变量：用关键字static声明。独立存在，位置放在某个类下，可以通过类名.静态变量名使用。程序启动运行该类时便常驻内存，无需初始化类再调用。

实例变量：类似于类的属性。需要初始化类才可调用。若该类未被再次使用，被GC回收，则不会存在于内存中。

（1）静态变量存储在静态存储区，而实例变量存储在堆中。

（2）静态变量前要加static关键字，而实例变量前不加。

（3）静态变量在类加载后便可以调用，而实例变量需要所属类初始化实例对象后才可调用。

（4）静态变量的生命周期取决于类的生命周期，而实例变量的生命周期取决于实例的生命周期。

Java面试题之集合类总结

发表于 2021-04-05 更新于 2021-04-06

集合类专题讲解+常见面试题汇总。

Java集合框架主要包括两种类型的容器，一种是集合Collection，存储一个元素集合，另一种是图Map，存储键值对映射。

所有的集合框架都包括以下内容：

  • 接口：代表集合抽象数据类型，如Collection、List、Set、Map等；
  • 实现（类）：集合接口的具体实现，是可重复使用的数据结构，如ArrayList、LinkedList、HashSet、HashMap等；
  • 算法：实现集合接口的对象里所执行的计算，如搜索和排序。

集合框架的优点：降低开发及维护成本，代码质量得到保障，增强复用性和可操作性。

常见面试题总结

◇◇◇◇◇◇◇◇◇◇◇◇◇◇◇

1.有哪些常见的集合？

（1）常见的集合有：

Collection
├List
│├LinkedList
│├ArrayList
│└Vector
│　└Stack
├Set
│├HashSet
│├TreeSet

Map
├Hashtable
├HashMap
└WeakHashMap

Collection是集合层级的根接口，Java平台不提供该接口的直接实现。

Set是一个不能包含重复元素的集合。

List是一个有序集合，可以包含重复元素。能够通过它的索引来访问任何元素，更像长度动态变换的数组。

Map是一个键值对映射对象。

（2）各类特点：

​ LIST

类名 顺序及可重复 底层实现 增查性能 安全性 容量扩充
ArrayList 排列有序可重复 数组 查询快，增删慢 线程不安全 当前容量*1.5+1
LinkedList 排列有序可重复 双向循环链表 查询慢，增删快 线程不安全 无
Vector 排列有序可重复 数组 查询快，增删慢 线程安全，效率低下 默认扩展一倍容量

​ SET

类名 顺序及可重复 底层实现 存储特点 备注
HashSet 无序不可重复 Hash表 存取速度快 内部是HashMap
TreeSet 无序不可重复 二叉树 排序存储 内部是TreeMap的SortedSet
LinkedHashSet 有序不可重复 Hash表，用双向链表记录插入顺序 \ 内部是LinkedHashMap

​ MAP

类名 可重复性 底层实现 安全性 允许null值
HashMap 键不可重复，值可重复 Hash表 线程不安全 允许key或value为null
HashTable 键不可重复，值可重复 Hash表 线程安全 key和value都不允许为null
TreeMap 键不可重复，值可重复 二叉树 \ \

2.ArrayList和LinkedList的区别

（1）ArrayList底层采用数组实现，LinkedList底层采用双向循环链表实现；

（2）ArrayList删改元素受位置影响，因此查询快，增删慢；LinkedList随机访问要移动指针，因此增删快，查询慢；

（3）ArrayList实现了RandomAccess接口，因此支持高效随即元素访问，而LinkedList不支持；

3.HashMap和HashTable的异同

相同点：都实现了Map接口（HashTable另实现了Dictionary抽象类）；

不同点：

（1）HashTable使用synchronized关键字修饰，是线程安全的；而HashMap是线程不安全的，不是同步的。

（2）HashMap的键值都可为null，而HashTable的键值都不可为null。

（3）HashMap的初始容量为16，扩容时容量翻倍；HashTable的初始容量为11，扩容时容量翻倍+1。

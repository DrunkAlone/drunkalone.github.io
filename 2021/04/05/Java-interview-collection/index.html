Java面试题之集合类总结（一）

发表于 2021-04-05 更新于 2021-04-10 分类于 面试题

集合类专题讲解+常见面试题汇总。

本篇主要内容：常见集合及Map相关。

Java集合框架主要包括两种类型的容器，一种是集合Collection，存储一个元素集合，另一种是图Map，存储键值对映射。

所有的集合框架都包括以下内容：

  • 接口：代表集合抽象数据类型，如Collection、List、Set、Map等；
  • 实现（类）：集合接口的具体实现，是可重复使用的数据结构，如ArrayList、LinkedList、HashSet、HashMap等；
  • 算法：实现集合接口的对象里所执行的计算，如搜索和排序。

集合框架的优点：降低开发及维护成本，代码质量得到保障，增强复用性和可操作性。

关于Iterator迭代器：迭代器并不是一个集合，它是一种用于访问集合的方法，为各类容器提供公共接口，可用于迭代ArrayList和HashSet等集合。

Collection是集合层级的根接口，Java平台不提供该接口的直接实现。

Set是一个不能包含重复元素的集合。

List是一个有序集合，可以包含重复元素。能够通过它的索引来访问任何元素，更像长度动态变换的数组。

Map是一个键值对映射对象。

2.ArrayList、LinkedList和Vector的区别

（1）ArrayList和Vector底层采用数组实现，LinkedList底层采用双向循环链表实现；

（2）ArrayList删改元素受位置影响，因此查询快，增删慢；LinkedList随机访问要移动指针，因此增删快，查询慢；

（3）ArrayList实现了RandomAccess接口，因此支持高效随即元素访问，而LinkedList不支持；

（4）ArrayList和LinkedList都是线程不安全的，Vector是线程安全的。

3.HashMap和HashTable的异同

相同点：都实现了Map接口（HashTable另实现了Dictionary抽象类）；

不同点：

（1）HashTable使用synchronized关键字修饰，是线程安全的；而HashMap是线程不安全的，不是同步的。

（2）HashMap的键值都可为null，而HashTable的键值都不可为null。

（3）HashMap的初始容量为16，扩容时容量翻倍；HashTable的初始容量为11，扩容时容量翻倍+1。

4.HashSet实现原理

HashSet实现Set接口，底层由HashMap实现，为哈希表结构，新增元素相当于HashMap的key，而value默认为一个固定的Object。

当有元素插入时，会计算元素的HashCode值，将元素插入到哈希表对应的位置中来。

它继承于AbstractSet，实现了Set、Cloneable、Serializable接口。

（1）HashSet继承AbstractSet类，获得Set接口大部分实现。

（2）HashSet实现Set接口，获取Set接口方法，可自定义也可继承。

（3）HashSet实现Cloneable，获得clone()方法。

（4）HashSet实现Serializable，可以被序列化。

5.HashSet去重原理

HashSet在添加元素时，会调用HashMap的put()方法进行对象唯一性判断。该方法会获取元素哈希值，判断key是否为同一个对象，判断key是否equas。

简单来说，第一步会先调用hashCode()方法获取该元素的哈希值，如果计算获取该元素的存储位置目前并没有任何元素存储，那么该元素可以直接存储；

反之进行第二步，调用equals()方法与该位置的元素再次比较，如果equals返回true，则不允许添加。

6.HashMap的数据结构

通常为数组+链表，而当链表长度大于等于8且数组长度大于64时，链表会被转化为红黑树，此时数据结构为数组+链表+红黑树。

7.HashMap的实现原理

HashMap是一个链表散列，基于hashing原理，使用put(key, value)方法存储对象至HashMap中，使用get(key)从HashMap中获取对象。

bucket：数组，该数组的每一个元素都是一个节点(entry)，该节点可引申为链表。同一个链表上节点的hash值相同。

  • put(key, value)：

    对key调用hashCode()方法进行hash值计算，随后将hash值与数组长度-1逐位相与，计算获得bucket存储索引值。

    判断该索引值是否有元素，如果为空，则直接存储；如果不为空，判断两个节点是否相同，相同会替换旧value，不相同则会以将该元素插入链表。

    当链表的长度长度大于等于8且数组长度大于64时，便会转换为红黑树。

    如果bucket桶满(容量16 * 加载因子0.75)，就会扩容。

    注意：在JDK1.8之前，链表采用头插法，JDK1.8采用尾插法。

  • get(key)：

    根据key的hash值计算索引，找到bucket节点位置，获取值对象。如果该位置只有一个元素，直接返回value，如果是链表结构，则循环链表获取该key的value。

8.如果HashMap中两个对象的HashCode相同？

  • put(key, value)：

    HashCode相同说明在bucket中的索引位置相同，此时会发生碰撞。这时进入链表数据结构，在该节点的头部或尾部插入元素。当链表长度达到8时，便会转换为红黑树。

  • get(key)：

    对key调用hashCode()方法获取hash值，根据hash值计算bucket索引，获取该位置对象。当两个对象hashCode相同，遍历该节点链表，调用keys.equals()方法找到链表中正确的节点，返回值对象。

9.HashMap为什么线程不安全？

高并发情形下，在调用put()或resize()方法时，可能导致某个index1下产生循环链表，此时如果get一个不存在的key，而该key的index正好等于index1，就会出现死循环和数据丢失。（在JDK1.8之后，并发执行put操作会出现数据覆盖的问题）

10.HashMap如何实现线程安全？

（1）使用Collection类的synchronizedMap方法包装，返回指定映射支持的同步；

（2）效仿ConcurrentHashMap的分段锁机制，对bucket进行分组，锁住要修改的部分来保证线程安全。

11.HashMap的扩容机制

当一个map填满了加载因子0.75*总容量16的时候，即size >= capacity × loadFactor时将会触发扩容机制，创建两倍大小的bucket数组，并将原来的对象放入新的数组中。

该过程称为rehashing，其调用hash方法找到新的bucket位置。

12.重新调整HashMap大小会产生什么问题？

重新调整HashMap大小时，存在条件竞争。如果两个线程都发现HashMap需要重新调整大小，它们会同时执行该操作。该过程中，存储在链表中的元素的次序会反过来，因为移动到新bucket中时，HashMap会使用头插法避免尾部遍历，此时条件竞争会导致死循环。

13.HashMap的键对象必要条件

任何对象都可以作为键，但需遵守equals()和hashCode()方法定义的规则，同时当对象插入后不可改变。

String这样的Wrapper类之所以适合作为键，是因为其用final修饰，保证不可变性，这样在计算hashCode的时候就不会产生键值变化，同时保证了线程安全，减少碰撞，提高性能。

14.HashMap的长度设置为什么是二次幂？

该长度服务于key映射到index的hash算法。位运算可以保证性能，效果和取模相同，这样设置长度可以保证二进制位数都是1，从而获得均匀的运算结果。

15.如何决定选用HashMap还是TreeMap？

当操作多为插入、删除、定位元素等，采用HashMap；

当操作多为对有序的key集合进行遍历，采用TreeMap。

16.HashMap为何采用红黑树结构？

红黑树是一种不严格的平衡二叉查找树，根节点和叶子节点定义为黑，红色节点不能相邻，每个节点到达可达叶子节点的所有路劲都应包括相同数目的黑色节点。

红黑树相当于排序数据，可以自动使用二分法进行定位，在插入删除时性能较高，同时较为稳定。HashMap通过使用红黑树，充分利用链表的内存使用率和红黑树的高级检索功能，将遍历复杂度从O(n)降低为O(log n)。

17.用ConcurrentHashMap替代HashTable？

可以。ConcurrentHashMap仅根据同步级别对map进行部分上锁，同步性能更好，但同时HashTable提供更强的线程安全性。

18.LinkedHashMap和TreeMap如何保证有序？

LinkedHashMap继承自HashMap，维护了一个链表，保证它内部元素的插入顺序，迭代时也按照插入的顺序迭代；

TreeMap按照key的自然顺序排序，可以继承Comparator接口实现。

19.Map的遍历方式

（1）keySet()方法，获取所有key的Set；

（2）values()方法，获取所有value的Collection；

（3）获取所有键值对集合，使用iterator迭代器遍历；

（4）获取所有键值对集合，使用for循环遍历。

20.HashMap解决碰撞（哈希冲突）的方法

（1）使用散列表来链接拥有相同哈希值的数据；

（2）使用hash函数降低碰撞概率，使数据分布更平均；

（3）引入红黑树进一步降低遍历的时间复杂度。

21.ConcurrentHashMap分段锁

  • JDK1.7中，ConcurrentHashMap是由Segment和HashEntry数组结构组成。Segment是一种可重入锁，HashEntry用于存储键值对数据。

    Segment数组将一个大table分割成多个小table进行加锁，即锁分离技术。每个Segment元素存储HashEntry数组+链表。

  • JDK1.8中，放弃Segment，直接使用数组+链表+红黑树的数据结构实现，并发控制使用Synchronized+CAS，线程安全且效率高。

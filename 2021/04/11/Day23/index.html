1.任务调度

题目描述：优秀的操作系统离不开优秀的任务调度算法。现在，有一台计算机即将执行n个任务，每个任务都有一个准备阶段和执行阶段。只有在准备阶段完成后，执行阶段才可以开始。同一时间，计算机只能执行一个任务的执行阶段，同时可以执行任意多个任务的准备阶段。请你设计一个算法，合理分配任务执行顺序，并输出完成所有任务的最少时间。

输入描述：第一行一个整数n表示任务的数量（1<=n<=5*10^4）

接下来n行每行两个整数a，b表示第i个任务的准备时长和执行时长。（1<=a,b<=10^9）

输出描述：仅一行一个整数，表示执行所有任务的最少时间。

示例：

是方才基于昨天的思路写出来的，不知道AC率多少。

主要思想是按照等待时间排序，初始化所需时间为第一个任务的等待时间+执行时间。

对于每一个任务，判断其等待时间是否大于所需时间，如果大于则所需时间加上该等待时间与上一步所需时间的差值。

遍历每个任务时都加上其执行时间，最终的结果即为最短时间。

2.等差植树

题目描述：小A的家门口有一排树，每棵树都有一个正整数的高度。由于树的高度不同，来到小A家的朋友都觉得很难看。为了将这些树高度变得好看，小A决定对其中某些树施展魔法，具体来说，每施展一次魔法，可以把一棵树的高度变成任意正整数（可以变高也可以变低）。

小A认为，这排树如果能构成等差为x的等差数列就好看了。但是小A不想施展太多次魔法，他想知道最少施展魔法的次数。

形式上来说，小A家门口一共有n棵树，第i棵树高度为ai。小A最后的目标为对于任意2≤i≤n，ai-ai-1=x。

输入描述：输入第一行包含两个正整数，n和x，含义如题面所示。

输入第二行包含n个正整数，第i个数的含义为第i棵树的高度ai。

范围：n≤10^5,1≤ai≤10^5，x≤1000

输出描述：输出包含一个正整数，即小A最少施展魔法的次数。

示例：

主要思想：该问题等价于，对于一些二维点[1,a1],[2,a2],…[i, ai]（前为树的序号，后为树的高度），最多有多少点能构成一条固定斜率为k（即题目中的等差值x）的直线y = kx + b。

因为斜率已知，每棵树的x和y已知，所以问题可以同化为b = y - kx，有多个b相同，b重复的最大次数即是在该施法过程中不需要变动的树的个数，于是需要施法的次数等于树的总数减去b重复的最大次数。

计算b重复的次数直接采用HashMap统计每个b的出现频率即可。

方法一、中心扩散法

主要思想：对于字符串的每一个字符，可以将其设为一个中心点。从中心点向两边扩散，如果两边字符相同，说明回文，继续扩散，反之则停止。对该字符串遍历完成之后，获取最长的子串即为所求串。

要注意的是，aba和abba分别作为奇数和偶数的回文串实例，所以对每个中心点，要考虑从自身出发、从自身和右侧字符出发两种情况，取二者之间较长的字符串返回比较。

主要思路：状态转移方程：

举例说明，对于字符串abcdedcghjk而言，如果要判断cdedc是回文串，其必须满足首尾字符相等'c' == 'c'且ded也为回文串这两个条件。

而对于边界条件，可以得知其长度的最小取值为1或2，即：

length = 1时：单个字符必然为回文串，确定为true，对应j = i；

length = 2时：头尾字符必须相等才为回文串，即charAt(i) == charAt(j)，其中j = i + 1。

如此以来，使用两层循环，一为子串的长度，二为子串的起始位置，再令dp为[初始位置，结束位置]的boolean二维数组，遍历即可获得结果。
